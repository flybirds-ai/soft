# 非研发人员战斗手册(名词解释)

## 前言

据说一个优秀的程序员和一个程序员的其中一个差异是：如何把专业性较强的技术原理用通俗易懂的语言描述出来。但即使如此，也无法做到把通俗易懂的技术原理解释一天说上一百遍~ 因此整理了一些和研发人员沟通时常见的技术术语的"通俗易懂"的解释，以更好的减少沟通成本。
